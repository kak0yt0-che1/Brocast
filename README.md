# Weather Bot

Телеграм-бот, который раз в день в выбранное время присылает погодную сводку по
городу (по умолчанию — Алматы). Прогноз берётся из бесплатного **Open-Meteo**,
текст собирается локально по правилам. Никаких платных API.

Есть два стиля сообщений:

- **Обычный** — вежливая сводка с советами (зонт, крем, одеться слоями).
- **Персона** — блатной матерный комментарий из набора фраз + факт о погоде.
  Включается флагом `PHRASE_MODE_ENABLED`.

Файл `app/phrases.json` содержит ненормативную лексику. Это осознанный стиль личного бота, его можно редактировать под себя.

## Как работает

```
раз в минуту планировщик проверяет, кому пора
        │
        ▼
Open-Meteo  →  сжатие прогноза  →  категория дня  →  фраза/совет  →  Telegram
 (weather.py)   (weather.py)       (phrases.py)      (advice.py)    (aiogram)
```

1. Скачиваем прогноз на сегодня и ужимаем его до компактной сводки
   (утро/день/вечер + ключевые показатели).
2. Навешиваем категорию: `heat / rain / cool_windy / freezing / chaotic / mild`.
3. В режиме персоны берём случайную фразу нужной категории из `phrases.json`
   и добавляем короткий факт; в обычном режиме собираем заголовок + советы.
4. Отправляем в Telegram. Дату последней отправки запоминаем, чтобы не слать
   дважды за день.

## Возможности

- Персональное время рассылки для каждого пользователя (свой часовой пояс).
- Inline-кнопки: «Сейчас», «Время», «Статус», «Выключить».
- Время задаётся пресетом, командой `/settime 07:45` или просто текстом `07:45`.
- Защита от повторной отправки в один день.
- Догон: если бот был выключен в нужную минуту, сводка всё равно уйдёт сегодня.
- Бэкофф при сбоях внешнего API (не спамит ретраями).

## Команды

| Команда | Действие |
|---|---|
| `/start` | регистрация и меню |
| `/now` | прислать сводку прямо сейчас |
| `/settime ЧЧ:ММ` | задать время ежедневной рассылки |
| `/timezone Area/City` | сменить часовой пояс (IANA, напр. `Asia/Almaty`) |
| `/status` | показать текущие настройки |
| `/stop` | выключить ежедневные уведомления |

## Запуск

Нужен Python 3.11+.

```powershell
cd "путь\к\weather bot"

# 1. зависимости
python -m venv .venv
.\.venv\Scripts\Activate.ps1        # Windows
pip install -r requirements.txt

# 2. секреты
copy .env.example .env              # затем впиши свой токен в .env

# 3. запуск
python -m app.main
```

Токен бота берётся у [@BotFather](https://t.me/BotFather). После запуска напиши
своему боту `/start`.

## Конфигурация (`.env`)

Все настройки — через переменные окружения. Обязательна только одна.

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | — (обязательно) | токен бота от BotFather |
| `PHRASE_MODE_ENABLED` | `true` | `true` — персона, `false` — вежливый режим |
| `CITY_NAME` | `Almaty` | название города в подписи |
| `LATITUDE` | `43.2389` | широта для прогноза |
| `LONGITUDE` | `76.8897` | долгота для прогноза |
| `DEFAULT_TIMEZONE` | `Asia/Almaty` | пояс по умолчанию для новых пользователей |
| `DATABASE_PATH` | `weatherbot.db` | путь к файлу SQLite |
| `LOG_LEVEL` | `INFO` | уровень логирования |

`.env` в гит не коммитится (см. `.gitignore`). В коде секретов нет.

## Структура проекта

```
app/
  main.py            точка входа: собирает зависимости, запускает бота и планировщик
  config.py          настройки из окружения (+ валидация на старте)
  logging_config.py  настройка логов
  weather.py         клиент Open-Meteo и сжатие прогноза в компактный вид
  advice.py          сборка текста по правилам (обычный режим + факт для персоны)
  phrases.py         выбор категории погоды и случайной фразы
  phrases.json       наборы фраз по категориям (данные, правятся вручную)
  keyboards.py       inline-клавиатуры меню
  handlers.py        команды и обработчики нажатий кнопок
  recommendations.py «дирижёр»: прогноз → текст → отправка, без дублей за день
  storage.py         SQLite: кто, во сколько, какой пояс, дата последней отправки
  validation.py      проверка пользовательского ввода (время, часовой пояс)
phrases.json пример  .env.example, requirements.txt, .gitignore
```

## Хранимые данные

Только необходимое для рассылки: `telegram_user_id`, `chat_id`, часовой пояс,
время уведомления, город, язык и дата последней отправки. Истории сообщений нет.

## Заметки

- Бот рассчитан на один запущенный экземпляр (SQLite, один поллер на токен).
- Время кнопками — только целые часы; получасы — командой или текстом.
- Чтобы бот работал постоянно, держи `python -m app.main` запущенным (или оформи
  как сервис/автозапуск).
